# ~help 命令 - 显示帮助

本模块定义帮助命令的输出内容。


## 目录
- 命令说明
- 执行模式适配
- 执行流程
- 输出模板

---

## 命令说明

```yaml
命令: ~help
类型: 直接执行类
功能: 显示帮助信息，无需确认或评估，直接输出帮助内容
触发: 用户输入包含 ~help 或 ~?
```

---

## 执行模式适配

> 规则引用: 按 G4 路由架构及 G5 执行模式规则执行

<mode_adaptation>
~help 模式适配规则:
1. 本命令为直接执行类，无需用户确认
2. 不受任何工作流模式影响
3. 检测上下文后直接输出帮助内容
4. 输出后保持待命状态，无状态变量设置
</mode_adaptation>

---

## 执行流程

### 步骤1: 上下文检测

<context_analysis>
上下文检测要点:
1. 检测用户输入是否携带上下文（疑问、困惑、请求等）
2. 如有上下文，准备简短回应用户的问题或情绪
3. 根据上下文调整下一步建议的针对性
</context_analysis>

```yaml
执行内容:
  - 检测用户输入是否携带上下文（疑问、困惑、请求等）
  - 如有上下文，准备简短回应用户的问题或情绪
  - 根据上下文调整下一步建议的针对性

目标: 友好引导，帮助用户快速上手
```

### 步骤2: 输出帮助信息

```yaml
执行内容:
  - 如步骤1检测到上下文，先简短回应
  - 按下方"输出模板"输出帮助内容
  - 按 G3 场景内容规则（直接回答）格式化输出

说明: 此命令为原子操作，无状态变量设置，输出后保持待命状态
```

- 输出包装（顶部状态栏/底部操作栏）以项目根目录 `AGENTS.md` 为准（SSOT）。
- 可用命令表以 `skills/helloagents/SKILL.md` 的“命令索引（SSOT）”为准（此处不再重复维护）。

- 状态描述：`使用帮助`
- 中间内容：

  - 使用方式（概述）：
